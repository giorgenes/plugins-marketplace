# File Generation Rules

When generating md files, prioritize creating individual note files instead
of a single long document with multiple sections.
Think of each file as being on section or subsection.

# File locations

Files should go under the folder of the stage/skill it belongs to.
For example, "understand" stage files should go under `understand/` folder.

# File naming format

Use the format `<stage>/<nn>-<title>.<format>`.

- stage: the skill stage used (ex: `understand/`).
- nn: the numerical count sequence of the file
- title: a short, easy to find description of what the file contains.

Prefix the file name with counter (`nn-`) in order to preserve the order
alphabetically.
For example: `understand/00-summary.md`, `understand/01-scope.md`, etc...
When adding new files always add 1 count to the latest count.
